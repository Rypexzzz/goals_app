# SETUP — Как развернуть и протестировать Aim

> Эта инструкция написана для человека, у которого нет опыта Android-разработки. Все шаги пронумерованы, каждое действие объяснено. Если в любой момент что-то не сходится — раздел [Что делать, если сломалось](#что-делать-если-сломалось) внизу.

---

## Что такое Aim и что в нём уже есть

**Aim** — приложение для целей и привычек. Всё хранится локально на телефоне, без облака и без интернета. В ветке `main` реализованы все 8 спринтов:

- **Сегодня** — чек-лист на день: регулярные задачи и привычки, просроченное.
- **Цели и задачи** — дерево задач до 5 уровней, дедлайны, markdown-описания, корзина (восстановление), архив.
- **Привычки** — частота (ежедневно / N раз в неделю / N раз в месяц / по дням недели), стрики, календарь.
- **Дашборд** — статистика и тепловые карты.
- **Уведомления** — напоминания о задачах и привычках.
- **Виджет «Сегодня»** на главном экране телефона.
- **Настройки** — темы (светлая / тёмная / системная), первый день недели, экспорт и импорт всех данных в JSON, архив, корзина.
- **Онбординг** при первом запуске.

Если цель — просто поставить приложение на телефон и пользоваться, иди сразу в раздел **§4 «Запуск на настоящем телефоне» → подраздел 4.4 «Установка для повседневного использования»**.

## 0. Глоссарий (зачем что нужно)

| Что | Зачем |
|-----|-------|
| **Android Studio** | Основной редактор кода + всё, что нужно для запуска Android-приложений. Это _IDE_ от Google, поставляется бесплатно. |
| **JDK** (Java Development Kit) | Android-приложения компилируются JDK. Современный AS уже привозит свой встроенный JDK, отдельно ставить не нужно. |
| **Android SDK** | Набор инструментов и системных образов Android разных версий. AS сам его скачает и установит. |
| **Gradle** | Система сборки. У нас уже подключён **Gradle Wrapper** — `./gradlew` в корне проекта. Не надо ставить Gradle отдельно. |
| **AVD** (Android Virtual Device) | Эмулятор телефона прямо на компе. Через AS им управляется. |
| **adb** | Утилита для общения с физическим телефоном по USB. Едет в комплекте с AS. |

## 1. Установка Android Studio

### 1.1. Скачать

Открыть https://developer.android.com/studio, нажать большую синюю кнопку **Download Android Studio**, принять условия.

Будет файл:
- **Windows:** `android-studio-xxxxx-windows.exe`
- **macOS:** `android-studio-xxxxx-mac.dmg` (Intel) или `android-studio-xxxxx-mac_arm.dmg` (Apple Silicon — M1/M2/M3)
- **Linux:** `android-studio-xxxxx-linux.tar.gz`

Версия должна быть **Ladybug** (2024.2.x) или новее — у нас AGP 8.7.3, более старая AS может не принять.

### 1.2. Установить

- **Windows:** запустить `.exe`, нажимать **Next → Next → Install**. Когда спросит про компоненты — оставить всё по умолчанию (Android Studio + Android Virtual Device).
- **macOS:** открыть `.dmg`, перетащить Android Studio в Applications, запустить из Launchpad.
- **Linux:** распаковать `.tar.gz`, например в `~/android-studio`, запустить `bin/studio.sh`.

### 1.3. Первый запуск (Setup Wizard)

При первом старте AS спросит:
1. **Import Settings?** — выбрать **Do not import settings**.
2. **Send usage data?** — на ваше усмотрение (на работу приложения не влияет).
3. **Install Type** — **Standard**.
4. **UI Theme** — Darcula или Light, не критично.
5. **Verify Settings** — AS покажет, что скачает (Android SDK Platform 35, Build Tools, Emulator). Это **5–10 ГБ**, скачается автоматически. Нажать **Next → Finish**.

Ждать, пока AS докачает всё. Это самый долгий шаг (10–30 минут в зависимости от интернета).

### 1.4. Проверить установку SDK

После настройки AS откроется **Welcome to Android Studio**.

Нажать (внизу справа) **More Actions → SDK Manager**.

В разделе **SDK Platforms** убедиться, что есть галка у **Android 15 (API 35)**. Если нет — поставить галку, нажать **Apply**, дождаться скачивания.

В разделе **SDK Tools** должны быть включены:
- Android SDK Build-Tools 35
- Android Emulator
- Android SDK Platform-Tools

Если чего-то нет — поставить галки и нажать **Apply**.

## 2. Открыть проект Aim

У вас есть два варианта.

### Вариант A — Через Android Studio (рекомендую новичкам)

1. На **Welcome**-экране нажать **Get from VCS** (Version Control System).
2. **URL:** `https://github.com/Rypexzzz/goals_app.git`
3. **Directory:** оставить по умолчанию (например, `~/AndroidStudioProjects/goals_app`).
4. **Clone**.
5. AS спросит **Trust Project?** — да, **Trust**.
6. AS начнёт **Gradle Sync** автоматически.

### Вариант B — Через терминал

```bash
git clone https://github.com/Rypexzzz/goals_app.git
cd goals_app
```

Ветка `main` уже содержит всё приложение (все 8 спринтов) — переключаться никуда не нужно.

Затем в AS: **File → Open → выбрать папку goals_app → OK**.

### 2.1. Что такое Gradle Sync?

После открытия AS внизу будет полоска **Sync in progress…** Это означает, что Gradle качает все зависимости (Compose, Hilt, Room и т.д.) — суммарно около **500 МБ** в первый раз. Может занять **5–15 минут**.

В правом нижнем углу будет крутиться спиннер. Дождаться, когда появится **Sync finished**.

### 2.2. Если Sync упал

См. раздел [Что делать, если сломалось](#что-делать-если-сломалось) → «Gradle Sync Failed».

## 3. Запуск на эмуляторе

### 3.1. Создать виртуальный телефон

1. В AS открыть **Tools → Device Manager** (или иконка телефона со стрелкой справа).
2. Нажать **+ Create Virtual Device**.
3. **Phone → Pixel 7** (или любой Pixel — стабильнее всего работают). **Next**.
4. **Recommended → Android 15 (VanillaIceCream)**. Если он не скачан — рядом будет ссылка **Download**, кликнуть и подождать (~1 ГБ).
5. **Next → Finish**.

### 3.2. Запустить эмулятор

В **Device Manager** рядом с созданным устройством нажать кнопку **▶** (Play).

Откроется окно эмулятора. Первый запуск **2–5 минут** (boot animation). Когда увидите домашний экран Android — готово.

### 3.3. Запустить приложение

В верхней панели AS:
1. Слева от ▶ должно быть **app**.
2. Справа от него — название вашего эмулятора (например, **Pixel 7 API 35**).
3. Нажать зелёную **▶ Run 'app'** (или Shift+F10 на Windows/Linux, Ctrl+R на macOS).

Что произойдёт:
- AS соберёт приложение (`./gradlew assembleDebug`) — первый раз **2–5 минут**.
- Установит APK на эмулятор.
- Запустит, откроется экран **Сегодня** с пустым состоянием.

✅ **Если увидели экран с надписью «Сегодня — чистый лист» и нижней навигацией на 4 пункта — поздравляю, Sprint 1 работает.**

## 4. Альтернатива — запуск на настоящем телефоне

Если у вас есть Android-телефон (13+) и кабель USB, это **быстрее эмулятора**.

### 4.1. Включить «Параметры разработчика»

1. На телефоне: **Настройки → О телефоне**.
2. 7 раз тапнуть по **Номер сборки**. Появится тост «Вы стали разработчиком».
3. Вернуться в **Настройки → Система → Для разработчиков**.

### 4.2. Включить USB debugging

В **Для разработчиков** включить:
- **Отладка по USB** (USB debugging).

### 4.3. Подключить кабель

1. Подключить телефон к компу USB-кабелем (по возможности — оригинальным).
2. На телефоне появится диалог **Разрешить отладку USB?** — **Разрешить** (можно поставить «Всегда»).
3. В AS, в селекторе устройств (верх) появится ваш телефон вместо эмулятора.
4. Нажать **▶ Run 'app'**.

✅ Через минуту приложение откроется на телефоне.

### 4.4. Установка для повседневного использования

Способ из §4.3 (**Run** из Android Studio) — это и есть установка: приложение **остаётся на телефоне как обычное**, кабель нужен только в момент установки. Отключил провод — пользуешься дальше без компа.

Если не хочешь каждый раз открывать Android Studio, собери APK один раз и поставь его вручную:

1. В терминале в папке проекта:
   ```bash
   ./gradlew assembleDebug
   ```
2. Готовый файл появится здесь: `app/build/outputs/apk/debug/app-debug.apk`.
3. Перенести APK на телефон любым способом: отправить себе в мессенджер/облако, или подключить телефон как флешку и скопировать.
4. На телефоне открыть этот файл (через «Файлы» или из мессенджера). Android попросит разрешить **установку из неизвестных источников** для этого приложения — разрешить.
5. Нажать **Установить**.

**Что важно знать:**

- Это **debug-сборка**. Для личного использования она полностью рабочая и ставится как обычное приложение. Это **не** подписанный release-APK для Google Play — но для себя он и не нужен.
- **Обновление до новой версии** (собрал заново, поставил тем же способом) — данные сохраняются: тот же `applicationId` (`com.aim.app`) и та же debug-подпись.
- **Удаление приложения или Wipe Data стирает базу.** Перед этим делай бэкап: **Настройки → экспорт в JSON**. Импорт — там же.
- Если приложение **крашится сразу при запуске** после переустановки — почти всегда это старая БД от прошлой версии. Удали приложение и поставь заново (данные перед этим лучше экспортировать).
- APK можно поставить и через кабель командой `adb install -r app/build/outputs/apk/debug/app-debug.apk` (флаг `-r` = переустановка с сохранением данных).

## 5. Сценарий полной проверки Sprint 1 (Foundation)

Запустить приложение и пройти по списку. Каждый пункт — отдельная галочка.

### 5.1. Навигация

- [ ] Стартовый экран — **Сегодня**. Видна нижняя панель с 4 иконками.
- [ ] Тап на **Цели** — переключается, заголовок «Цели», виден пустой экран и FAB «Новая цель».
- [ ] Тап на **Привычки** — пустое состояние «Тут будут твои привычки».
- [ ] Тап на **Дашборд** — «Графики появятся позже».
- [ ] Перейти назад в **Сегодня** — нижняя панель остаётся видимой.

### 5.2. Настройки и темы

- [ ] На любой из 4 вкладок тапнуть иконку **шестерёнки** справа вверху → открывается экран **Настройки**.
- [ ] Видны секции **Внешний вид** (3 темы) и **Данные** (Архив, Корзина).
- [ ] Тап **Тёмная** — вся UI мгновенно перекрашивается в тёмную тему.
- [ ] Тап **Светлая** — обратно.
- [ ] Тап **Как в системе** — тема следует системной. Свернуть приложение, в настройках Android поменять системную тему, вернуться — приложение перекрасилось.
- [ ] Нажать **←** (стрелка назад) — возвращаемся туда, откуда зашли. Нижняя панель снова видна.

### 5.3. Шрифт Inter

- [ ] Заголовки экранов отрисованы шрифтом **Inter** (узнаваемый чёткий гротеск).
- Если шрифт не «применился» — это нормально на старом эмуляторе без Google Services. На реальном устройстве и стандартном эмуляторе он подтянется автоматически.

## 6. Сценарий полной проверки Sprint 2 (Цели и задачи)

Это главный тест — полный жизненный цикл данных. Идти строго по порядку.

### 6.1. Создание цели

- [ ] Открыть вкладку **Цели**.
- [ ] Нажать FAB **Новая цель** (внизу справа).
- [ ] Открылся bottom sheet **Новая цель**.
- [ ] В поле названия ввести: `Стать сильнее физически`.
- [ ] Нажать большую эмодзи слева → открылся **Эмодзи Picker** → выбрать категорию **Тело** → выбрать `💪`. Sheet с эмодзи закрылся.
- [ ] Нажать **Дедлайн → выбрать дату** (через ~3 месяца) → **Применить**.
- [ ] В поле **Описание** ввести любой markdown: `**Главное**: тренировки 3 раза в неделю.`
- [ ] Переключить **Превью** — увидеть, что **Главное** выделено жирным.
- [ ] Переключить обратно в **Редактировать**.
- [ ] Нажать **Сохранить**.
- [ ] Sheet закрылся. На экране **Цели** появилась карточка с эмодзи, названием, дедлайном и описанием.

### 6.2. Глубокое дерево задач

- [ ] Тап на карточку цели → открылся **GoalDetailScreen**.
- [ ] Видна шапка с эмодзи / названием / дедлайном / описанием (markdown отрендерен).
- [ ] Под заголовком — секция **Задачи**, пока пусто.
- [ ] FAB **Добавить задачу** → открывается форма **Новая задача**.
- [ ] Создать задачу 1-го уровня: `Силовые тренировки`. Сохранить.
- [ ] В дереве появилась строка с чекбоксом, названием, кнопкой `+` и `⋮`.
- [ ] Тап на `+` справа от задачи → bottom sheet, родитель = эта задача → создать: `Жим лёжа`. Это 2-й уровень.
- [ ] У `Жим лёжа` тап `+` → создать `Программа Madcow`. 3-й уровень.
- [ ] У `Программа Madcow` тап `+` → создать `Сделать тестовый цикл`. 4-й уровень.
- [ ] У `Сделать тестовый цикл` тап `+` → создать `Записать веса в день А`. 5-й уровень — **последний**.
- [ ] У `Записать веса в день А` (depth=4) **кнопки `+` НЕТ** — это ожидаемо, ограничение глубины работает.
- [ ] Раскрытие/сворачивание: тап на «стрелочку вниз» у `Силовые тренировки` → дерево сворачивается. Ещё тап — разворачивается. Анимация поворота стрелки заметна.

### 6.3. Чекбокс с пружиной

- [ ] Тап по чекбоксу `Жим лёжа` → галочка появляется с пружинной анимацией (легко bouncy), название перечёркивается.
- [ ] Ещё тап — снимается.

### 6.4. Drag-n-drop внутри уровня

- [ ] Создать 3 задачи первого уровня: `A`, `B`, `C`.
- [ ] У каждой строки справа — иконка с 6 точками (ручка перетаскивания).
- [ ] Зажать ручку у `C` и перетащить наверх — порядок меняется в реальном времени.
- [ ] Отпустить — порядок сохраняется в БД.
- [ ] Закрыть и снова открыть цель → порядок сохранился.
- [ ] **Проверка ограничения:** попытаться перетащить задачу 1-го уровня между подзадачами 2-го уровня — отпускание игнорируется, элемент возвращается на место. Это правильное поведение.

### 6.5. Завершение цели с диалогом

- [ ] В GoalDetail тапнуть **⋮** справа вверху → **Отметить выполненной**.
- [ ] Появляется диалог **Также отметить задачи?** (потому что есть незавершённые подзадачи).
- [ ] Кнопка **И задачи** → диалог закрылся, все задачи в дереве стали с галочками, статус цели = Завершена.
- [ ] **⋮ → Вернуть в активные** — задачи остаются завершёнными, но сама цель снова IN_PROGRESS.

### 6.6. Фильтры

- [ ] Вернуться на экран **Цели**.
- [ ] Сверху чипы: **Все · Активные · Завершённые · Архивные**. Сейчас выбран **Активные**.
- [ ] Тап **Завершённые** — список пуст (мы вернули цель в активные).
- [ ] Создать вторую цель «Дочитать Войну и мир». В её детальном экране через **⋮ → Отметить выполненной** (без задач — диалога не будет).
- [ ] Вкладка **Завершённые** — видна эта цель.
- [ ] **Все** — видны обе цели.

### 6.7. Архив

- [ ] У `Стать сильнее физически` через **⋮ → Архивировать**.
- [ ] Цель исчезла из вкладок **Активные / Все / Завершённые**.
- [ ] Фильтр **Архивные** — цель там.
- [ ] **Настройки → Архив целей** (или через **⋮ → Архив** на экране Цели) → видна та же цель.
- [ ] Тап **Вернуть из архива** → цель снова появляется в Активных.

### 6.8. Корзина (soft delete + restore)

- [ ] Открыть `Стать сильнее физически` → **⋮ → Удалить**.
- [ ] Появляется диалог подтверждения. **Подтвердить**.
- [ ] Цель исчезла из Активных. Экран Goal Detail закрылся автоматически (back-pop).
- [ ] **Настройки → Корзина** (или через **⋮ → Корзина**) → видна цель в корзине с датой удаления.
- [ ] Тап **Восстановить** → цель снова в Активных. Все её задачи и подзадачи на месте.
- [ ] Удалить ещё раз → в корзине → **Удалить навсегда** (с диалогом) → исчезает окончательно. Это пройдёт через FK CASCADE — задачи тоже удалятся.

### 6.9. Задача в корзине (индивидуально)

- [ ] В Goal Detail у одной из задач тап **⋮** → **Удалить**.
- [ ] Задача исчезает из дерева.
- [ ] **Настройки → Корзина** → виден элемент с типом «Задача» и подписью «Из цели «...»».
- [ ] **Восстановить** → задача возвращается в дерево.

### 6.10. Хлебные крошки в TaskDetail

- [ ] В дереве задач тапнуть по тексту задачи (не на чекбокс/⋮) → откроется **TaskDetailScreen**.
- [ ] Сверху видна цепочка: **🎯 Цель → Задача 1 → Задача 2 → Текущая задача**.
- [ ] Тап на любой родительский элемент → переходим на тот уровень (либо обратно к Goal).

✅ **Если все 60 чек-боксов выше зелёные — Sprint 2 принят.**

## 7. Запуск автоматических тестов

В терминале (на macOS/Linux — обычный, на Windows — Git Bash или PowerShell):

```bash
cd путь/к/goals_app
./gradlew test
```

Должно пройти **~15 unit-тестов**. В отчёте увидите что-то вроде:
```
BUILD SUCCESSFUL in 2m 12s
8 actionable tasks: 8 executed
```

Подробный HTML-отчёт: `app/build/reports/tests/testDebugUnitTest/index.html` — открыть в браузере.

Хотите только домен-тесты:
```bash
./gradlew :app:testDebugUnitTest --tests "com.aim.app.domain.*"
```

## 8. Что делать, если сломалось

### «Gradle Sync Failed» / красные ошибки после открытия проекта

1. **File → Invalidate Caches… → Invalidate and Restart**.
2. Если не помогло — удалить папки `~/.gradle/caches/` и `.gradle/` в корне проекта, заново синкнуть.
3. Проверить, что AS использует **JDK 17** (Embedded JDK):
   **File → Settings → Build, Execution, Deployment → Build Tools → Gradle → Gradle JDK**: должен быть `jbr-17` или `Embedded JDK 17`.

### Эмулятор не запускается / тормозит

- **Windows:** проверить, что в BIOS включена виртуализация (VT-x / AMD-V) и установлен Intel HAXM / Hyper-V.
- **macOS Apple Silicon:** обязательно скачать ARM-образ системы (в Device Manager → Create Device → Other Images → ARM).
- Маломощный комп — взять более простой образ: вместо API 35 ставить **API 34** и Pixel **3a** (меньше памяти).

### `./gradlew` ругается «Permission denied» (macOS/Linux)

```bash
chmod +x gradlew
./gradlew test
```

### Приложение установилось, но при открытии падает (crash)

В AS открыть **View → Tool Windows → Logcat** → отфильтровать по `com.aim.app`. Стэктрейс покажет где. Самые частые причины:
- **Hilt** не сгенерил DI — пересобрать **Build → Clean Project → Rebuild Project**.
- **Room schema mismatch** — на устройстве осталась БД от прошлой версии: на эмуляторе сделать **Wipe Data** в Device Manager, на телефоне удалить приложение перед переустановкой.

### Кнопка **Run** серая / нет селектора устройств

- Не доделан Gradle Sync. Дождаться завершения внизу окна.
- Если в селекторе **No devices** — открыть Device Manager и явно запустить эмулятор.

### На «Сегодня» / «Привычки» / «Дашборд» FAB не работает

Это **ожидаемо** для Sprint 1 и Sprint 2. Эти экраны — заглушки, которые наполнятся в Спринтах 3 и 4.

## 9. Что не сделано (это нормально)

В `main` реализованы **все 8 спринтов** — основной функционал ТЗ готов (см. список в начале файла). Сознательно **отложено** на потом (детали в `docs/DECISIONS.md`, ADR-0024) — это полировка, без неё приложение полностью рабочее:
- Shared-element-анимация перехода между списком целей и деталкой.
- Swipe-жесты и drag-n-drop в части списков.
- Скелетоны-плейсхолдеры на время загрузки.
- Настройка размера шрифта.
- Экран конфигурации виджета (`WidgetConfigurationActivity`).
- Baseline Profile для ускорения холодного старта.

Разделы §5 и §6 ниже — подробные тест-сценарии Спринтов 1 и 2. Остальные функции (привычки, «Сегодня», уведомления, дашборд, виджет, экспорт/импорт) тестируются по аналогии — открыл вкладку и прошёлся по очевидным действиям.

## 10. Совет напоследок

Перед каждой большой сессией тестирования делайте на эмуляторе/телефоне **Wipe Data** (Device Manager → значок 3 точки рядом с устройством → Wipe Data). Это стирает данные приложения и БД, и вы получаете чистый старт, как у нового пользователя.

Если в процессе тестирования найдёте баг или странность — открыть issue в этом репозитории с описанием шагов и скриншотом из эмулятора (**Cmd/Ctrl+S** в окне эмулятора делает скриншот).
